# Contributing to Task Flow Monitor

First off, thank you for taking the time to contribute! Contributions are what make the open-source community such an amazing place to learn, inspire, and create.

All types of contributions are encouraged and valued. See below for details on how to get started.

---

## 🛠️ How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue in our repository. Use our Bug Report template and include:
* A clear title and description.
* Step-by-step instructions to reproduce the issue.
* Expected vs. actual results.
* Screenshots or console log outputs.

### Suggesting Enhancements
We welcome ideas for new features! To suggest an enhancement:
1. Open a Feature Request issue.
2. Explain the use case and why it would benefit other users.
3. Describe the implementation concept.

### Code Contributions
Ready to write some code? Here is the workflow:
1. **Fork the Repository**: Create a personal copy on GitHub.
2. **Clone & Setup**: Clone your fork locally and run standalone manual tests using `node test.js`.
3. **Write Code**: Ensure your files match clean-coding practices.
4. **Test**: Run verification tests locally.
5. **Commit**: Keep your commit messages descriptive (e.g., `feat: add custom logs view to dashboard`).
6. **Submit PR**: Open a Pull Request back to our `main` branch.

---

## 💻 Local Development Guidelines

* **Pure Vanilla JavaScript**: Keep the codebase zero-dependency! Do not add external npm packages unless absolutely required for primary core functionality.
* **Code Formatting**: Run Prettier to format files before committing.
* **Responsive Styling**: Ensure dashboard edits look beautiful and responsive on both desktop and mobile viewports.

Thanks for keeping our project premium!
